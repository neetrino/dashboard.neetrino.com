# Функциональность сброса статуса синхронизации Bitrix24

## Описание
Добавлена новая функция для сброса статуса синхронизации в блоке аналитики "Статус синхронизации". Это позволяет администратору сбросить информацию о последней отправке данных в текущем месяце, делая возможной повторную отправку.

## Что добавлено

### 1. Кнопка "Сбросить" в админ-интерфейсе
- Расположена рядом с кнопкой "Синхронизировать сейчас"
- Имеет вторичный стиль (серая кнопка)
- Содержит tooltip с подсказкой о функционале
- При клике показывает подтверждающий диалог

### 2. Обработчик сброса статуса
**Файл:** `bitrix24.php`
**Метод:** `reset_status_handler()`
- Проверяет права пользователя
- Проверяет nonce для безопасности
- Вызывает публичный метод `reset_sync_status()`
- Перенаправляет пользователя с уведомлением о результате

### 3. Публичный метод сброса
**Файл:** `bitrix24.php`
**Метод:** `reset_sync_status()`
- Удаляет следующие опции:
  - `neetrino_bitrix24_last_send_date`
  - `neetrino_bitrix24_last_send_status` 
  - `neetrino_bitrix24_next_send_date`
- Логирует операцию
- Возвращает boolean результат

### 4. Уведомления
- Успешный сброс: "Статус синхронизации успешно сброшен!"
- Ошибка сброса: "Ошибка при сбросе статуса синхронизации!"
- Уведомления автоматически исчезают через 5 секунд

### 5. JavaScript обработчик
- Подтверждающий диалог с предупреждением
- Блокировка кнопки во время выполнения
- Индикация процесса ("Сбрасываем...")
- Создание и отправка формы с nonce

## Принцип работы

1. Пользователь нажимает кнопку "Сбросить"
2. Появляется диалог подтверждения
3. При подтверждении отправляется POST-запрос
4. Сервер проверяет права и nonce
5. Удаляются опции статуса синхронизации
6. Пользователь перенаправляется с уведомлением
7. После сброса `should_send_this_month()` вернет `true`

## Безопасность

- Проверка прав пользователя (`manage_options`)
- WordPress nonce для предотвращения CSRF атак
- Подтверждающий диалог для предотвращения случайного нажатия
- Логирование всех операций

## Использование

После сброса статуса:
- Система будет считать, что в текущем месяце отправка не производилась
- Станет возможной повторная отправка данных в том же месяце
- При следующем автоматическом запуске cron задача выполнится
- Ручная синхронизация также будет работать

## Техническая интеграция

Функция интегрирована с существующей системой:
- Использует те же проверки прав доступа
- Следует паттернам WordPress admin-post
- Совместима с существующей логикой `should_send_this_month()`
- Не нарушает работу других функций модуля
